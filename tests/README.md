# Тесты

Запуск (БД не нужна — тесты идут на in-memory SQLite):

```bash
pip install -r requirements.txt
pytest
```

## Что покрыто

- **test_payments_api.py** — Payments API: внутренний перевод, получение
  платежа, валидация суммы, проверка владения счётом списания (свой/чужой/
  команда), нехватка средств, авторизация.
- **test_interbank_api.py** — `/interbank/receive`: зачисление,
  идемпотентность по `transfer_id`, аутентификация, отрицательная сумма,
  несуществующий счёт, запрет self-as-sender.
- **test_payment_service.py** — логика `PaymentService`: валидации,
  внутренний перевод, межбанк (успех/возврат, капитал) с замоканным HTTP.
- **test_vrp_api.py** — VRP: лимит количества платежей, лимит суммы за
  период, лимит на платёж, проверка владельца согласия.

## Как устроено

`conftest.py` поднимает приложение через ASGI-транспорт, подменяет `get_db`
на сессию SQLite и перенаправляет туда же логирующий middleware. ARRAY-колонки
рендерятся как JSON только для создания схемы (тесты в них не пишут).
